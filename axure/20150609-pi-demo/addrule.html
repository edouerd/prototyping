Basically the trigger structure flows in this way:

 

1. Select Trigger

2. Define Trigger Phrase (target, action, target)

3. The phrase is dependent on the trigger, so it could change drastically

 

Action Flow:

 

1. Select an Action Type (in this case we only have 'Send a Message'

2. Select Action Subset (send an sms/notification/email)

3. Define Target Segment — in this case, I have questions regarding targeting unique people of the 'Customer' segment — i used the phrase 'instance customer', but there's  gotta be a better way to handle this.

4. Add another Action — you can imagine this to be the 'And...' clause in this logic, i.e.

 

If a customer walks into swimwear, send a message AND send an email AND notify an employee.
